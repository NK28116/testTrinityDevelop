# 基本設計書

## 1. システム構成

```
[Client (OS: Windows/macOS/Linux)] --(実行)--> [Python Interpreter] --(標準出力)--> [Console/File]
```

クライアントOS上でPythonインタプリタが実行され、標準出力に結果を出力します。ログはコンソールまたはファイルに出力されます。

## 2. 主要コンポーネント

### 2.1 フロントエンド

*   **画面表示:** 標準出力 (`print()`) を使用して、コンソールにテキストを表示します。GUIフレームワークは使用しません。

### 2.2 バックエンド

*   **ビジネスロジック:** 画面に表示するテキストを定義し、ログを出力する処理を実装します。
*   **ログ出力:** `logging` モジュールを使用して、コンソールまたはファイルにログを出力します。

### 2.3 データ層

*   **データ:** 特に永続化するデータはありません。表示するテキストはコード内にハードコードします。

## 3. データ構造

今回は、データ構造を定義する必要はありません。

## 4. API定義

APIは定義しません。

## 5. 技術選定理由

*   **プログラミング言語:** Python
    *   要件定義書で提案されている通り、学習コストが低く、クロスプラットフォームに対応しているため。
    *   標準ライブラリが充実しており、小規模なアプリケーション開発に適しているため。
*   **フレームワーク:** 特になし (標準ライブラリのみ使用)
    *   要件定義書で提案されている通り、依存関係を最小限に抑え、環境構築を簡素化するため。
*   **IDE:** Visual Studio Code
    *   要件定義書で提案されている通り、拡張機能が豊富で、Python開発に適しているため。
*   **バージョン管理:** Git (GitHub)
    *   要件定義書で提案されている通り、バージョン管理を容易に行い、開発履歴を管理するため。

## 6. セキュリティ設計

セキュリティは特に考慮しません。最小限の動作確認を目的とするためです。

## 7. 非機能要件への対応

*   **パフォーマンス:** Pythonの標準ライブラリのみを使用し、シンプルなコードで実装することで、起動時間とリソース消費を最小限に抑えます。
*   **スケーラビリティ:** 今回のアプリケーションはスケーラビリティを考慮しません。
*   **可用性:** 今回のアプリケーションは可用性を考慮しません。
